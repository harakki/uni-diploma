# Дипломный проект "Разработка веб-сервиса для чтения графической литературы"
Я начал разработку дипломной работы бакалавра. Моя идея - веб-сервис для чтения манги под названием "MangeDex". Главной фишкой сервиса будет продвинутая система персональных рекоммендаций (сервис должен будет анализировать как предпочитаемый пользователем контент, так и паттерны его поведения, сопоставлять параметры с пользователями, похожими на него, чтобы предложить максимально релевантный контент, рассчитав, при этом, процент совместимости (то есть, система, похожая на алгоритмы рекоммендаций Netflix, Amzon и Spotify)). 

## Технологический стек
Бекенд будет написан на Java 25 (со вставками Kotlin 2.2, где необходимо) и Spring Boot 4. Данные храниться в PostgreSQL и медиа в MinIO. Системой аутентификации будет Keycloak. Брокер сообщений Kafka. Все сервисы будут представлять собой Docker-контейнеры, разворачиваемые как инстансы Kubernetes.
Фронтенд будет написан на Vue.js (необходимо учесть, что я вообще не знаком с данным фреймворком и фронтенд-разработкой в целом).

## Архитектруа
Архитектура приложения будет представлять собой модульный монолит.

## Функционал
Сервис должен предоставлять:

1. Персональные подборки тайтлов
2. Персональные списки пользователей ("Прочитано", "Любимое", "Брошено" и т.д.) с возможностью делиться по ссылке (как это можно делать с плейлистами в музыкальных сервисах)
3. Возможность оставлять комментарии под подборками, тайтлами, главами, и страницами тайтлов
4. Профили пользователей, авторов и издательств, на уведомления об из
менениях которых можно подписаться
5. Систему персональных вознаграждений и достижений по результатам каких-либо действий (например, достижение за прочтение 1000 тайтлов, достижение за вход 365 дней подряд, достижение за проведение в общей сложности 24 часов за чтением, прочтение определенного тайтла полностью и т.д.)
6. Продолжение чтения тайтла с места остановки
7. Выбор глав произведения
8. Поиск по фильтрам (название, теги, жанры, количество глав, возврастной рейтинг, тип, формат выпуска, статус выпуска тайтла/перевода) и сортировка результатов (по рейтингу, по релевантности, по названию, по просмотрам, по количеству глав, по дате релиза, по дате обноления/добавления и т.д.)
9. Списки подборок на главной странице (например, популярно сейчас, последние обновления и т.д.)
10. Меню уведомлений и пуш-уведомления по обновлениыям тайтлов, авторов, издателей, пользователей и т.д.
11. PWA-приложение
12. Удобную читалку и интерфейс сервиса с возможностью гибкой настройки параметров внешнего вида и конфиденциальности (включая настройку внешнего вида профиля пользователей)
13. Возможность оставлять приватные (личные) заметки к страницам глав тайтла с интерфейсом для из просмотра по тайтлу
14. Интерфейс для добавления команд переводчиков, авторов, издательств, тайтлов, глав и т.д.
15. Ежегодную статистику пользователя (в стиле Spotify Wrapped)
